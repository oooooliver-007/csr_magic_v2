# 事件 CRUD — 功能规格

## 概述
管理端事件（Event）的完整增删改查功能。事件是多个活动的容器，如"2026春季CSR月"。

## 所属信息
- 模块：event
- 前置依赖：auth（Admin 角色）

## 用户故事
- 作为管理员，我希望创建 CSR 事件来组织一批相关活动
- 作为管理员，我希望编辑事件信息以保持内容准确
- 作为管理员，我希望控制事件在员工端的可见性

## 页面内容

### 事件列表页
- 表格列：事件名称、类型、时间范围、活动数量、状态、操作
- 搜索框（按名称搜索）
- 状态筛选下拉
- 「新建事件」按钮
- 操作列：查看（View 抽屉）、编辑（Edit 抽屉）、删除（确认弹窗）
- 分页控件

### 创建/编辑事件（侧边抽屉）
- 事件名称（必填，≤200字）
- 描述（富文本）
- 类型：线下 / 线上 / 混合
- 开始/结束时间（日期选择器）
- 可见地区（多选）
- 可见角色（多选）
- 是否在用户端展示（开关）
- 封面图上传（≤500KB，支持预览和清除）

## 交互说明
- 新建/编辑保存成功 → Toast 提示 + 自动刷新列表
- 删除 → 二次确认弹窗 → 确认后删除 → Toast 提示
- 搜索实时筛选（防抖 300ms）
- 表单校验失败 → 字段下方行内红色提示
- 显示开关切换 → 即时生效，Toast 提示

## 验收标准
- [ ] 事件列表正确展示，支持分页
- [ ] 搜索和状态筛选正常工作
- [ ] 新建事件包含所有必填字段验证
- [ ] 编辑事件正确预填现有数据
- [ ] 删除事件有二次确认
- [ ] 封面图上传支持预览、清除、大小限制提示
- [ ] 显示开关正常切换
- [ ] 移动端：表格切换为卡片列表布局
