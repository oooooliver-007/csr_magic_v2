# 登录/注册 — 功能规格

## 概述
员工通过用户名密码进行身份验证，进入系统。未注册用户可自行注册账号。

## 所属信息
- 模块：auth
- 前置依赖：无

## 用户故事
- 作为员工，我希望通过用户名和密码登录系统，以便访问 CSR 活动
- 作为新员工，我希望注册一个账号，以便开始参与 CSR 活动

## 页面内容

### 登录页
- 公司 Logo + 系统名称（CSR Magic）
- 用户名输入框
- 密码输入框
- 「登录」按钮
- 「去注册」跳转链接

### 注册页
- 姓名（必填）
- 用户名（必填，唯一）
- 密码（必填，≥6位）
- 所在地区（下拉选择）
- 性别（可选）
- 「注册」按钮
- 「去登录」跳转链接

## 交互说明
- 登录成功 → 跳转首页（员工端 `/`，管理员 `/admin`）
- 登录失败 → 行内错误提示（用户名或密码错误），不弹窗
- 注册成功 → 自动登录并跳转首页
- 注册失败 → 行内提示具体原因（用户名已存在等）
- 未登录访问受保护页面 → 重定向到登录页
- 密码输入框支持显示/隐藏切换

## 验收标准
- [ ] 登录页正确渲染，包含用户名、密码输入框和登录按钮
- [ ] 注册页包含所有必填字段和验证
- [ ] 登录成功后根据角色跳转到对应首页
- [ ] 登录失败显示行内错误提示
- [ ] 注册成功后自动登录
- [ ] 用户名重复时给出明确提示
- [ ] 未登录状态下访问受保护路由自动跳转登录页
- [ ] 页面在桌面端和移动端均正常显示
