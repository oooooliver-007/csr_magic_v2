# 家属同行报名 — 功能规格

## 概述
扩展现有报名流程，支持员工在报名时携带家属一同参加活动。家属作为报名记录的附属信息存储，不需要独立账号、独立审核或独立通知。管理员在创建活动时可勾选"允许携带家属"并配置每人最多携带的家属数量。

## 所属信息
- 模块：participation
- 前置依赖：signup（已实现）、activity-crud（管理端活动表单需扩展）

## 用户故事
- 作为员工，我希望在报名活动时填写携带的家属信息（姓名 + 关系），让家属也能参加
- 作为员工，我希望知道当前活动是否允许带家属、最多能带几位
- 作为管理员，我希望在创建活动时按需开启或关闭家属同行功能，并控制每人携带上限
- 作为管理员，我希望在审核报名时看到员工申请携带的家属列表
- 作为管理员，我希望活动名额能正确反映实际到场人数（员工 + 家属合并占用名额）

## 员工端功能

### 报名时携带家属
- 仅当 `activity.allowFamily = true` 时，报名表单显示"家属信息"区块
- 区块顶部提示："本活动允许携带家属，家属将与您一起占用活动名额"
- 每位家属填写：
  - **姓名**（必填，文本）
  - **关系**（必填，下拉选择：配偶 / 子女 / 父母 / 其他）
- 操作：
  - "+ 添加家属"按钮（达到 `maxFamilyPerUser` 上限时禁用）
  - 每行右侧"删除"按钮
  - 显示当前/上限计数（如 "2/5"）
- 校验：
  - 姓名空白的行视为未填，不计入提交
  - 关系必须为枚举值之一
  - 家属总数不超过 `maxFamilyPerUser`
- 提交时家属列表与表单数据一同发送

### 我的参与记录回显
- "我的参与"卡片在活动名下方显示"携带 N 名家属"（仅 N>0 时）
- 重新提交（驳回后再报名）时回填原家属列表，可编辑

## 管理端功能

### 活动创建/编辑
- 表单中新增"允许携带家属"开关（默认关闭）
- 开关打开后展开"每人最多携带家属数"输入框（数字 ≥ 1，留空 = 不限）
- 关闭"允许携带家属"时清空 `maxFamilyPerUser` 输入

### 参与审核详情
- 审核详情区在 formData 字段下方新增"家属列表"区块
- 显示："家属信息（共 N 人）"
- 列表项：`姓名（关系）`，例如 `张小明（配偶）`、`张小红（子女）`
- 顶部副标题："本次报名占用名额：N + 1（含本人）个"

### 参与列表（明细页）
- "参与内容摘要"列末尾追加 `· +N 家属`（仅 N>0 时）
- 不增加额外筛选条件（保持简洁）

### 名额计算
- 活动剩余名额计算方式：`maxParticipants - SUM(1 + 家属数量)`
- 报名时校验：员工本人 + 携带家属数量 ≤ 当前剩余名额
- 名额满或不足时报名按钮置灰，提示"剩余名额不足以容纳您和家属"

## 交互说明
- 关闭活动 `allowFamily` 后，已报名的历史记录中的家属信息仍保留并可查看（不删除）
- 退出活动时随主参与记录一同删除家属信息
- 报名通知（站内通知）内容不变，无需提及家属
- AI 海报生成不为家属生成独立海报；本期不修改海报 prompt

## 验收标准
- [ ] 管理员创建活动可勾选"允许携带家属"，并设置每人上限
- [ ] 仅 `allowFamily = true` 时报名表单显示家属信息区块
- [ ] 员工可添加/删除多个家属（≤ `maxFamilyPerUser`），姓名 + 关系两个字段
- [ ] 关系字段限定为 SPOUSE / CHILD / PARENT / OTHER 枚举
- [ ] 家属姓名为空的行不被计入提交
- [ ] 报名提交后家属信息正确持久化
- [ ] 名额校验合并员工 + 家属总数，超过 `maxParticipants` 时报名失败
- [ ] 已审核报名记录中的家属信息在管理端审核详情可见
- [ ] 个人中心"我的参与"显示"携带 N 名家属"
- [ ] 旧活动 `allowFamily` 默认 false，行为完全不变
- [ ] 旧参与记录 `familyMembers` 为空，前端不显示家属区块
- [ ] 关闭活动 allowFamily 后，已有报名的家属信息仍可查看
- [ ] 重新提交（RE_SUBMITTED）时回填原家属列表
- [ ] 单元测试覆盖：开关关闭拒绝带家属 / 超上限拒绝 / 名额合并校验
- [ ] Playwright E2E：报名带家属全流程 / 管理端审核查看家属
