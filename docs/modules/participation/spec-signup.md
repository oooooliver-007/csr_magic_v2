# 报名/退出 — 功能规格

## 概述
员工通过活动详情页提交报名表单参与活动，也可在报名后退出。管理员在参与明细页查看和审核报名记录。

## 所属信息
- 模块：participation
- 前置依赖：activity-detail（报名入口在活动详情页）

## 用户故事
- 作为员工，我希望快速报名感兴趣的活动
- 作为员工，我希望在需要时退出已报名的活动
- 作为管理员，我希望审核员工的报名记录（通过/驳回）

## 员工端功能

### 报名
- 在活动详情页填写对应模板的表单字段
- 点击「立即报名」提交
- 提交后状态变为"待审核"（PENDING）
- 名额满时报名按钮置灰

### 退出
- 已报名的活动显示「退出活动」按钮
- 点击后二次确认 → 退出成功
- 仅 PENDING 状态可退出

## 管理端功能

### 参与明细页
- 筛选条件：事件、活动、员工、状态、时间范围
- 列表列：员工姓名、活动名称、报名时间、参与内容摘要、状态、审核人、操作
- 点击行 → 展开详情（完整表单内容、附件图片）
- 操作：「通过」/「驳回」（驳回必须填写原因）
- 系统自动发通知给员工

## 交互说明
- 报名成功 → Toast 提示 + 自动发送站内通知
- 退出活动 → 二次确认 → 成功后更新状态
- 审核通过 → 更新状态 + 通知员工
- 审核驳回 → 填写原因 → 更新状态 + 通知员工
- 员工在个人中心可查看审核状态

### 驳回后重新提交
- 被驳回的记录在活动详情页展示驳回原因 + 「修改后重新提交」按钮
- 点击后展开表单（可基于旧数据继续修改）
- 提交后调用 `POST /api/v2/participations/{id}/resubmit`
- 状态从 REJECTED 变为 RE_SUBMITTED，进入管理员再次审核队列
- 仅记录所有者在状态为 REJECTED 且活动未结束时可调用

## 验收标准
- [ ] 员工可通过表单报名活动
- [ ] 报名后状态正确变为 PENDING
- [ ] 名额满时报名按钮正确置灰
- [ ] 已报名用户可退出活动（仅 PENDING 状态）
- [ ] 管理端参与明细筛选正常工作
- [ ] 管理端可展开查看完整报名内容
- [ ] 审核通过/驳回功能正常
- [ ] 驳回时必须填写原因
- [ ] 审核后自动发送站内通知
- [ ] 移动端管理端：表格切换为卡片（含展开详情和审核按钮）
- [ ] 员工可对被驳回的记录点击「修改后重新提交」并更新表单
- [ ] 重新提交后状态变为 RE_SUBMITTED，清空驳回原因/审核信息
- [ ] 非 REJECTED 状态或非记录所有者调用 resubmit 接口时返回错误
