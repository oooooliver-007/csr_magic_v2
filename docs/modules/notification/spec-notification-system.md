# 站内通知 — 功能规格

## 概述
站内通知系统，自动向员工发送报名确认、审核结果、活动提醒等通知。支持未读角标、已读标记和全部已读。

## 所属信息
- 模块：notification
- 前置依赖：auth（认证）

## 用户故事
- 作为员工，我希望及时收到活动报名和审核结果的通知
- 作为员工，我希望一眼看到有多少未读通知

## 页面内容

### 通知铃铛（Header 组件内）
- 铃铛图标 + 未读数角标（红色，超过 99 显示"99+"）
- 点击铃铛 → 展开下拉列表（最近 5 条）
- 每条通知：标题、内容摘要、时间
- 底部「查看全部」链接

### 通知列表页
- 所有通知列表（按时间倒序）
- 未读通知高亮显示
- 点击通知 → 标记已读 + 跳转对应页面
- 「全部标记已读」按钮

## 通知触发场景
- 报名成功 → 自动发送确认通知
- 审核通过 → 通知员工
- 审核驳回 → 通知员工（含驳回原因）
- 活动即将开始 → 提醒已报名员工

## 交互说明
- 未读角标实时更新（轮询或页面刷新时）
- 点击通知自动标记已读
- 通知内容支持变量模板（如 `{{activity_name}}`）

## 验收标准
- [ ] 铃铛图标正确展示未读数
- [ ] 超过 99 显示"99+"
- [ ] 点击铃铛展开最近 5 条通知
- [ ] 通知列表页按时间倒序展示
- [ ] 未读通知有视觉区分
- [ ] 点击通知自动标记已读并跳转
- [ ] 全部标记已读功能正常
- [ ] 报名/审核操作正确触发通知
