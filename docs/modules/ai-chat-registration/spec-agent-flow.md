# Agent 对话流程 — 功能规格

## 概述
Qwen Agent 根据活动模板类型动态调整对话策略，逐步收集报名所需字段，确认后提交。对话深度随活动复杂度自动调整。

## 所属信息
- 模块：ai-chat-registration
- 前置依赖：chat-ui（对话界面）

## 用户故事
- 作为员工，我希望 AI 引导我完成报名，只问必要的问题
- 作为员工，我希望在确认前看到报名摘要，确保信息正确

## 对话流程

```
系统开场白（根据活动类型定制）
→ Agent 逐步收集必填字段（根据模板动态提问）
→ 所有字段收齐 → Agent 展示确认摘要
→ 用户回复「确认」→ 调用 API 写入 DB → 成功提示
→ 用户要求修改 → Agent 更新对应字段 → 重新展示摘要
```

## 对话深度（按活动类型）

| 模板类型 | 对话轮次 | Agent 开场示例 |
|---------|---------|--------------|
| 签到活动 | 1 轮 | "你好！我来帮你完成签到，一键确认即可。确认参加吗？" |
| 基础活动 | 1-2 轮 | "你好！需要留下参与备注吗？直接输入内容，或回复「跳过」。" |
| 捐赠活动 | 3-4 轮 | "你好！我来帮你完成捐赠报名，先告诉我你计划捐赠多少金额？" |
| 志愿者活动 | 3-4 轮 | "你好！请告诉我你计划参与的服务时长（如：3小时）。" |
| 自定义活动 | 按字段数 | 按管理员配置字段逐一询问 |

## 错误处理
- 用户输入无效内容（如捐赠金额填了文字）→ Agent 友好提示重新输入
- 用户偏离话题 → Agent 引导回报名流程
- 网络错误提交失败 → Agent 提示"提交遇到问题，请稍后重试"，保留已填信息

## 确认摘要格式

```
我将为你完成以下报名，请确认：
• 活动：《春季植树活动》
• 捐赠金额：200 元
• 留言：希望能帮助更多人
回复「确认」提交，或告诉我需要修改的内容
```

## 验收标准
- [ ] Agent 根据活动类型发送定制化开场白
- [ ] 简单活动（签到/基础）1-2 轮完成
- [ ] 复杂活动（捐赠/志愿者）3-4 轮完成
- [ ] 自定义活动按字段数动态调整
- [ ] 所有字段收齐后展示确认摘要
- [ ] 用户确认后正确提交报名
- [ ] 用户要求修改时正确更新字段
- [ ] 无效输入有友好的重新输入提示
- [ ] 提交失败保留已填信息
