# 对话界面 — 功能规格

## 概述
AI 对话报名的前端界面，提供类似聊天应用的交互体验，用户通过自然语言与 AI Agent 对话完成活动报名。

## 所属信息
- 模块：ai-chat-registration
- 前置依赖：activity-detail（从活动详情页进入）

## 用户故事
- 作为员工，我希望通过对话方式完成报名，比填表更自然轻松

## 入口
- 活动详情页 →「AI 对话报名」按钮（与传统「立即报名」并排）

## 页面布局
- 左侧：当前活动信息卡（活动名、时间、类型）
- 右侧：全屏对话窗口

## 页面内容

### 对话窗口
- 消息气泡：用户消息右对齐（蓝色），AI 消息左对齐（灰色）
- AI 消息有打字机动画效果
- 底部输入框 + 发送按钮
- 「切换表单模式」链接（允许随时切换到传统表单）
- 成功提交后显示绿色成功卡片 + 「查看报名详情」跳转

### 活动信息卡
- 活动名称
- 活动类型标签
- 活动时间
- 需要收集的字段列表

## 交互说明
- 进入页面 → AI 自动发送开场白
- 用户输入文字 → 发送 → AI 回复（带打字机动画）
- 发送中 → 输入框禁用 + loading 指示
- 「切换表单模式」→ 跳转到传统报名表单（保留已收集信息）
- 网络错误 → AI 提示"网络异常，请重试"

### 草稿自动保存
- 已收集的字段实时保存（sessionStorage）
- 意外关闭后重新进入 → "上次对话未完成，是否继续？"
- 选择「继续」→ 恢复进度；选择「重新开始」→ 清空

## 验收标准
- [ ] 对话窗口正确渲染消息气泡
- [ ] 用户消息右对齐，AI 消息左对齐
- [ ] AI 消息有打字机动画
- [ ] 发送消息后正确调用 API 并展示回复
- [ ] 切换表单模式链接正常工作
- [ ] 草稿自动保存和恢复功能正常
- [ ] 成功提交后显示绿色成功卡片
- [ ] 移动端：顶部活动信息卡（可折叠）+ 全屏对话框
