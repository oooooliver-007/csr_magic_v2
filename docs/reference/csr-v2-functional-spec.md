## 一、用户角色

| 角色 | 说明 | 访问入口 |
|------|------|---------|
| **普通员工（User）** | 公司员工，浏览和参与 CSR 活动 | 用户端 Web（桌面 + 移动端） |
| **管理员（Admin）** | HR 或 CSR 团队，管理活动和用户 | 管理后台（桌面 + 移动端） |

## 二、用户端（Employee Web App）

### 2.1 信息架构

```
用户端
├── 登录 / 注册
├── 首页（Home）
├── 活动（Activities）
│   ├── 活动列表
│   └── 活动详情 + 报名（传统表单方式）
├── AI 对话报名（Chat Registration）⭐ AI Feature 1
├── 个人中心（My Profile）
│   ├── 参与记录
│   └── 个人设置
└── AI 海报工作台（Poster Studio）⭐ AI Feature 2
```

### 2.2 两个 AI 功能入口对比

| | AI 对话报名 | AI 海报工作台 |
|--|------------|------------|
| **触发时机** | 活动进行中（报名阶段） | 活动结束后 |
| **核心模型** | Qwen（通义千问）大语言模型 | Wan 2.7 Image Pro 图像生成模型（通义万相） |
| **用户输入** | 自然语言对话 | 照片 + 文字提示词 |
| **输出** | 活动报名记录（存 DB） | 个性化活动海报（图片） |
| **入口** | 活动列表页 / 活动详情页的「AI 对话报名」按钮 | 个人中心 / 首页快捷入口 |

### 2.2 页面详情

---

#### 页面 1：登录 / 注册

**目的**：员工身份验证，进入系统。

**登录页内容**：
- 公司 Logo + 系统名称
- 用户名 / 密码输入框
- 登录按钮
- 跳转注册链接

**注册页内容**：
- 姓名、用户名、密码、所在地区（下拉）、性别（可选）
- 注册按钮

**交互说明**：
- 登录成功 → 跳转首页
- 登录失败 → 行内错误提示（不弹窗）
- 未登录访问其他页面 → 重定向登录页

---

#### 页面 2：首页（Home）

**目的**：让员工快速了解当前 CSR 活动动态和自己的参与情况。

**页面内容**：
- 顶部：欢迎语（"Hi，[姓名]"）+ 个人参与统计卡片（总参与活动数、累计志愿时长、累计捐赠额）
- 活动推荐区：最新 / 即将开始的活动卡片（3-4 张）
- 最近参与区：员工自己最近的活动记录（时间轴样式）
- 快速入口：「生成我的专属海报」按钮（引导至 AI 海报工作台）

**活动卡片包含**：
- 活动封面图
- 活动名称
- 所属事件名称
- 活动类型标签（捐赠 / 志愿者 / 签到 / 基础）
- 开始 ~ 结束时间
- 当前报名人数
- 「查看详情」按钮

---

#### 页面 3：活动列表（Activities）

**目的**：浏览全部可参与的 CSR 活动。

**页面内容**：
- 搜索栏（按活动名称搜索）
- 筛选条件：活动类型、状态（进行中 / 即将开始 / 已结束）、所属事件
- 活动卡片网格（同首页卡片样式）
- 分页 / 无限滚动

**空状态**：无活动时展示插画 + "暂无可参与的活动" 文案

---

#### 页面 4：活动详情（Activity Detail）

**目的**：查看活动完整信息并完成报名。

**页面内容**：
- 活动封面大图
- 活动名称、类型标签、状态标签
- 所属事件（可点击）
- 时间：开始时间 ～ 结束时间
- 活动描述（富文本）
- 参与表单（根据活动类型不同）：
  - **基础活动**：文字说明（可选填）
  - **捐赠活动**：捐赠金额（必填）+ 留言（选填）
  - **志愿者活动**：服务时长（必填）+ 照片上传（最多5张）
  - **签到活动**：自动记录时间，可选上传照片
  - **自定义活动**：按管理员配置的表单字段显示
- 底部操作栏：「立即报名」/ 「已报名 · 退出」/ 「活动已结束」

**状态说明**：
- 未报名 → 显示报名表单 + 报名按钮（若有名额上限，显示「剩余名额：X」）
- 名额已满 → 报名按钮置灰，显示「名额已满」提示
- 已报名 → 显示已填信息 + 「退出活动」选项
- 审核驳回 → 显示驳回原因 + 「修改后重新提交」按钮（可重新编辑表单并提交）
- 活动结束 → 只读模式，显示参与记录

---

#### 页面 5：个人中心（My Profile）

**目的**：查看个人 CSR 参与历史和贡献数据。

**页面内容**：
- 个人信息卡（头像、姓名、部门/地区）
- 贡献统计：参与活动数、志愿时长、捐赠总额（数字卡片）
- **Tab 切换**：
  - **Tab 1 — 参与记录**：
    - 每条记录：活动名、参与时间、状态（审核中 / 已通过 / 已驳回）、活动类型图标
    - 状态为「已驳回」时显示橙色标签 + 「查看原因并重新提交」链接
    - 可点击查看完整详情
    - 右上角「导出记录」按钮 → 导出当前用户全部参与记录为 PDF 或 CSV
  - **Tab 2 — 我的海报**：
    - 展示用户历史生成的所有海报（卡片网格）
    - 每张卡片：海报缩略图、对应活动名、生成时间、「下载」「分享」按钮
    - 右上角「去生成海报」快捷入口
- 个人设置：修改昵称、真名、密码（需验证当前密码）

#### 页面 5b：AI 对话报名（Chat Registration）⭐ AI Feature 1

**目的**：用户通过自然语言对话完成活动报名，无需手动填写表单。

**入口**：活动详情页 → 「AI 对话报名」按钮（与传统「立即报名」并排展示）

**页面布局**：
- 左侧：当前活动信息卡（活动名、时间、类型）
- 右侧：全屏对话窗口

**对话流程**：

```
系统开场白
"你好！我来帮你完成《[活动名称]》的报名。
这是一个[活动类型]活动，我需要收集以下信息：[字段列表]
我们开始吧，请告诉我..."
        ↓
Agent 逐步收集必填字段
（根据活动模板动态提问）
        ↓
所有字段收齐后，Agent 展示确认摘要
"我将为你完成以下报名，请确认：
• 活动：《春季植树活动》
• 捐赠金额：200 元
• 留言：希望能帮助更多人
回复「确认」提交，或告诉我需要修改的内容"
        ↓
用户回复「确认」→ 调用 API 写入 DB → 成功提示
用户回复修改意见 → Agent 更新对应字段 → 重新展示摘要
```

**对话深度按活动复杂度自动调整**（所有类型均可用）：

| 模板类型 | 对话轮次 | Agent 开场示例 |
|---------|---------|--------------|
| 签到活动 | **1 轮** | "你好！我来帮你完成《XXX》的签到，一键确认即可。确认参加吗？" |
| 基础活动 | **1-2 轮** | "你好！需要留下参与备注吗？直接输入内容，或回复「跳过」。" |
| 捐赠活动 | **3-4 轮** | "你好！我来帮你完成捐赠报名，先告诉我你计划捐赠多少金额？" |
| 志愿者活动 | **3-4 轮** | "你好！请告诉我你计划参与的服务时长（如：3小时）。" |
| 自定义活动 | **按字段数** | 按管理员配置字段逐一询问 |

> 原则：简单活动让 AI 对话比传统表单更快；复杂活动体现对话收集的价值。

**草稿自动保存**：
- 用户在对话中已收集的字段实时保存（`sessionStorage` 本地缓存）
- 意外关闭页面后重新进入，显示「上次对话未完成，是否继续？」提示
- 用户选择「继续」→ 恢复对话进度；选择「重新开始」→ 清空草稿

**对话 UI 规范**：
- 消息气泡样式（用户消息右对齐，AI 消息左对齐）
- AI 消息有打字机动画效果
- 底部输入框 + 发送按钮
- 「切换表单模式」链接（允许用户随时切换到传统表单）
- 成功提交后显示绿色成功卡片，提供「查看报名详情」跳转

**错误处理**：
- 用户输入无效内容（如捐赠金额填了文字）→ Agent 友好提示重新输入
- 网络错误提交失败 → Agent 提示"提交遇到问题，请稍后重试"，保留已填信息

#### 页面 6：AI 海报工作台（Poster Studio）⭐ AI Feature 2（✅ 已实现）

**目的**：员工基于自己的活动参与记录，生成个性化 CSR 纪念海报。

**触发条件**：用户有参与记录（从已参与活动中选择）。

**页面布局**：单栏垂直布局，上方输入区 + 下方历史画廊。

**输入区内容（✅ 实际实现）**：

1. **选择活动**（下拉）：从“我已参与的活动”中选择 1 个（调 participationApi.myParticipations）
2. **风格选择**（6 种预设风格卡片）：
   - 简约现代 / 水彩手绘 / 3D 立体 / 卡通插画 / 国潮 / 写实摄影
   - 点击即选，高亮显示当前选中风格
3. **自定义提示词**（可选）：
   - 文本输入框，用户可补充描述希望的海报内容
   - 与风格选择拼接后传入 AI 服务
4. **“生成海报”按钮**

**生成中状态（✅ 已实现）**：
- 按钮变为 loading 状态（disabled + spinner）
- 提示语：“AI 正在为你创作专属海报，请稍等约 20 秒...”
- 前端每 3 秒轮询 `GET /posters/{taskId}/status`
- 遇到 401/403 自动停止轮询，提示“登录已过期，请重新登录后再试”

**结果展示（✅ 已实现）**：
- 海报图片展示（从 AI 服务 StaticFiles 返回 URL）
- 自动刷新历史画廊

**历史海报画廊（✅ 已实现）**：
- 网格卡片布局，显示历史生成的所有海报
- 每张卡片：海报缩略图 + 活动名 + 生成时间
- 点击卡片 → Lightbox 大图预览（点击打开，再次点击关闭，fadeIn + scaleIn 动画）

│   └── 用户详情

### 3.2 页面详情

---

#### 页面 A：数据看板（Dashboard）

**目的**：管理员一眼看到 CSR 项目的整体运营状态。

**内容区块**：
- **顶部统计卡片**（4个）：总活动数、累计参与人次、累计捐赠额、本月新增参与
- **参与趋势折线图**：近 12 个月每月参与人次
- **活动类型分布饼图**：各类型活动占比
- **最活跃员工 Top 10 列表**：按参与次数排名
- **近期活动状态列表**：显示正在进行的活动及参与情况

---

#### 页面 B：事件管理（Events）

**目的**：管理 CSR 大事件（Event = 多个活动的容器）。

**列表页**：
- 表格：事件名称、类型、时间范围、活动数量、状态、操作
- 搜索框 + 状态筛选
- “新建事件”按钮

**创建/编辑事件（侧边抽屉或独立页）**：
- 事件名称（必填）
- 描述（富文本）
- 开始/结束时间（日期选择器）
- 类型：线下 / 线上 / 混合
- 可见地区（多选）
- 可见角色（多选）
- 是否在用户端展示（开关）
- 封面图上传

---

#### 页面 C：活动管理（Activities）

**目的**：管理具体 CSR 活动（隶属于某个事件）。

**列表页**：
- 表格：封面缩略图（40×40）、活动名称、所属事件、类型/模板、时间、状态、参与人数、操作
- 按事件筛选、按状态筛选
- **操作列**：
  - 👁 **View**：打开只读详情抽屉（封面图、所有字段、参与人数进度条）；底部可快捷跳转 Edit
  - ✏️ **Edit**：打开预填表单抽屉，保存调 `PUT /activities/{id}`

**创建/编辑活动**（侧边抽屉）：
- 所属事件（下拉，必填）
- 活动名称
- 活动模板选择：基础 / 捐赠 / 志愿者 / 签到 / 自定义
- 开始/结束时间
- 人数上限（可选，留空表示不限）
- 描述
- **封面图上传**：点击上传区域选择本地图片，自动转换为 base64 存入数据库（≤ 500KB；超过大小显示错误提示）；支持实时预览 + 清除
- **自定义模板**：字段配置器（字段名 + 字段类型：文字/数字/图片/开关）（🔲 Phase 3）

> 原则：简单活动让 AI 对话比传统表单更快；复杂活动体现对话收集的价值。

**草稿自动保存**：
- 用户在对话中已收集的字段实时保存（`sessionStorage` 本地缓存）
- 意外关闭页面后重新进入，显示“上次对话未完成，是否继续？”提示
- 用户选择“继续”→ 恢复对话进度；选择“重新开始”→ 清空草稿

**对话 UI 规范**：
- 消息气泡样式（用户消息右对齐，AI 消息左对齐）
- AI 消息有打字机动画效果
- 底部输入框 + 发送按钮
- “切换表单模式”链接（允许用户随时切换到传统表单）
- 成功提交后显示绿色成功卡片，提供“查看报名详情”跳转

**错误处理**：
- 用户输入无效内容（如捐赠金额填了文字）→ Agent 友好提示重新输入
- 网络错误提交失败 → Agent 提示“提交遇到问题，请稍后重试”，保留已填信息

---

#### 页面 D：参与明细（Participation）

**目的**：查看和审核员工的活动参与记录。

**筛选条件**：事件、活动、员工、状态（待审核 / 已通过 / 已驳回 / 重新提交）、时间范围

**列表列**：员工姓名、活动名称、报名时间、参与内容摘要、状态、审核人、操作

**操作**：
- 点击行 → 展开详情（查看完整表单内容、附件图片）
- “通过”/ “驳回”审核（驳回时必须填写驳回原因，系统自动发通知给员工）
- **批量审核**：列表支持复选框多选，顶部出现“批量通过”按钮，二次确认后批量处理
- “导出 Excel”按钮（导出当前筛选结果）

**重新提交处理**：
- 员工修改后重新提交的记录，状态变为“重新提交”，在列表中优先级高亮
- 管理员可看到修改前后的内容对比（diff 视图）

---

#### 页面 E：用户管理（Users）

**目的**：管理系统内所有员工账号。

**列表页**：

## 五、关键用户流程图

### 员工：传统表单报名

```
登录 → 首页 → 活动列表 → 活动详情 → 「立即报名」→ 填写表单 → 提交
                                                              ↓
                                                    等待管理员审核
                                                              ↓
                                               收到站内通知（审核结果）
```

### 员工：AI 对话报名（Qwen Agent）⭐

```
活动详情页 → 「AI 对话报名」按钮 → 打开对话窗口
                                        ↓
                              Agent 开场白（介绍活动 + 开始收集信息）
                                        ↓
                              多轮对话（用户自然语言输入 → Agent 提取字段）
                                        ↓
                              所有必填项收齐 → Agent 展示确认摘要
                                        ↓
                              用户回复「确认」→ 报名写入 DB → 成功提示
                              （或用户要求修改 → 重新收集 → 再次确认）
```

### 员工：生成 AI 海报（Wan 2.7 Image Pro）⭐（✅ 已实现）

```
员工端导航「Poster」入口 → 海报工作台
→ 选择已参与的活动 → 选择风格（6种）→ 可选填写自定义提示词 → 「生成海报」
→ 等待生成（约20秒，前端每3秒轮询）→ 查看海报 → 历史画廊 Lightbox 大图预览
```

### 管理员：创建新一期活动

```
登录 → 事件管理 → 新建事件 → 填写事件信息 → 保存
         ↓
      活动管理 → 新建活动 → 选择所属事件 → 选择模板 → 填写信息 → 保存
         ↓
      活动上线（用户端可见）
```

### 管理员：审核参与记录

```
参与明细 → 筛选「待审核」→ 逐条查看（展开表单 + 附件）
→ 单条：点击「通过」或「驳回」（驳回填写原因）
→ 批量：勾选多条 → 「批量通过」→ 二次确认 → 完成
→ 系统自动发站内通知给对应员工
```

### 员工：驳回后重新提交

```
收到驳回通知 → 个人中心 Tab1「参与记录」→ 找到橙色「已驳回」记录
→ 「查看原因并重新提交」→ 打开表单（预填旧数据）→ 修改 → 提交
→ 状态变为「重新提交」→ 等待管理员再次审核
```

### 员工：查看通知

```
顶部导航铃铛（红色角标）→ 点击展开最近通知列表
→ 点击单条通知 → 跳转对应活动/参与记录详情
→ 「查看全部」→ 全部通知页（可标记已读 / 全部已读）
```
